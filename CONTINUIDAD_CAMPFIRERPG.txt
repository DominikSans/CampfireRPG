CampfireRPG - Continuidad de trabajo
Fecha base del ultimo estado: 2026-03-31

1. Vision actual del plugin
- El plugin sigue enfocado en campfires como sistema cooperativo de descanso, soporte y bonus por grupo.
- Se mantuvo el soporte para clanes como feature extra opcional.
- Si el servidor no tiene plugin de clanes, el plugin sigue funcionando normal.
- Si el jugador no tiene clan, el plugin sigue funcionando normal.
- UltimateClans y otros hooks siguen siendo un extra, no un requisito.

2. Cambios importantes realizados
- Se reorganizo config.yml para que sea mas entendible y con comentarios.
- Se corrigio la creacion/copia de lang/es.yml y lang/en.yml al iniciar.
- Se creo un sistema de runtime overrides para que la GUI no destruya comentarios ni el orden de config.yml.
- El debug ahora persiste si se activa desde comando o desde GUI.
- Se optimizo el registro de campfires para evitar reescaneos completos de chunks en eventos simples.
- Se rehizo el sistema del menu admin para que sea editable desde gui.yml.
- El nuevo gui.yml soporta:
  - slot
  - slots
  - rangos como 10-14
  - items-decorate
  - item_model
  - validacion de slots duplicados o fuera de rango
- Se rediseño la GUI para parecerse mas a la imagen enviada:
  - marco perimetral decorativo
  - cabeza de piglin como informacion central
  - opciones en el centro
  - flechas abajo para navegar entre paginas
  - boton de cerrar centrado abajo
- Se hizo el editor de perfiles mas flexible:
  - ya no depende solo de normal y soul
  - ahora recorre perfiles disponibles
- Se restauraron los defaults de clanes como feature extra opcional.

6. Flujo que se sigue en cada respuesta cuando hay cambios
- Primero reviso el codigo y la configuracion real antes de decidir cambios.
- Si el cambio es grande, aplico cambios directamente y luego valido con build.
- Cada vez que el cambio merece nueva version, aumento la version en gradle.properties segun criterio:
  - patch: fixes pequenos
  - minor: cambios funcionales visibles sin romper compatibilidad
  - major: cambios que rompen compatibilidad o estructura base
- Despues corro:
  - ./gradlew.bat build
- Si el build pasa, hago git:
  - git add .
  - git commit -m "mensaje"
  - git push origin HEAD

7. Regla acordada de versionado
- Si hay un cambio pequeno: 4.4.0 -> 4.4.1
- Si hay un cambio funcional visible: 4.4.0 -> 4.5.0
- Si hay un cambio rompedor o reestructuracion fuerte: 4.4.0 -> 5.0.0
